            Wiring help — GPS
            
    
              
  • Plug into any Pi USB port; prefer the stable /dev/serial/by-id/… entry.
    • 
              
  • Stock u-blox speaks NMEA by default — select Serial (wired); choose u-blox M9N (UBX) for richer 10 Hz UBX mode (the driver configures the receiver itself on start).
    • 
              
  • Permission error? Run: sudo usermod -a -G dialout $USER then re-login.

            Wiring help — Compass (HWT901B-TTL)
            
    
              
  • TTL device — needs a USB-UART adapter: VCC→5V, GND→GND, sensor TX→adapter RX (crossed).
    • 
              
  • Factory baud: 9600.
    • 
              
  • Mount flat, X toward the bow, away from the motor's magnetic field (>50 cm ideally).
    • 
              
  • Re-run Sensor calibration after mounting.
    •

            Wiring help — Motor (vanchor firmware)
            
    
              
  • One or two Arduino Nanos over USB (engine = thrust, steering = azimuth) at 115200.
    • 
              
  • Wiring, pin maps and BOM: firmware/README.md.
    • 
              
  • The probe only listens for the firmware's A/E feedback lines — it never sends a motor command; nothing can spin.
    • 
              
  • Alternative: the helm PCB tunnels the same protocol over I²C at bus 1, address 0x42 (motor_port: i2c:1:0x42) — select the I²C option below.
    • 
            

          

          

            Battery shunt (INA226) — I²C
            
    
              
  • I²C bus 1 (Pi pins 3/5), default address 0x40, shunt in the battery NEGATIVE lead.
    • 
              
  • Set battery.shunt_ohms in vanchor.yaml to your shunt's rating.
    • 
              
  • Use the Probe button above with an I²C option selected to verify.
    •
