vanchor.sim.fossen

A higher-fidelity 3-DOF (surge-sway-yaw) maneuvering boat model.

This is a drop-in alternative to vanchor.sim.boat.Boat. It exposes the same interface (state, step, truth) but instead of a first-order speed lag plus a kinematic yaw rate it integrates a proper rigid-body + hydrodynamic model in body-fixed coordinates [u, v, r] (surge, sway, yaw rate).

The structure is inspired by the MIT-licensed Fossen "otter" USV from cybergalactic/PythonVehicleSimulator:

M * nu_dot + C(nu_r) * nu_r + (D_lin + D_quad(nu_r)) * nu_r = tau

where nu = [u, v, r] are body velocities, M = M_rb + M_a is the rigid-body plus added mass, C(nu) is the Coriolis-centripetal matrix (the body-frame / rotating-hull coupling + added-mass Munk moment -- NOT the negligible planetary Earth-rotation Coriolis effect), D_lin / D_quad are linear and quadratic damping, and tau is the generalized force/moment from the thruster plus the aerodynamic wind force. The hydrodynamic terms act on the velocity through the water nu_r = nu - nu_c (so a current advects the hull and is felt as drag), while the kinematics integrate the absolute velocity nu. Wind enters tau as a quadratic aerodynamic force / yaw moment (not a fixed leeway), so leeway and weathervaning emerge from the force balance.

Thrust mapping: this models a single steerable trolling motor mounted at the bow, at a signed longitudinal offset thruster_x_m from the centre of gravity (positive = forward / bow). The motor produces a thrust of magnitude T = command.thrust * max_thrust_n (negative = reverse) directed along its steered axis delta = command.steering * radians(max_steer_angle_deg):

Fx = T * cos(delta)        # surge
Fy = T * sin(delta)        # sway (+ = starboard)
N  = thruster_x_m * Fy - thruster_y_m * Fx   # yaw from both lever arms

tau = [Fx, Fy, N]

The key consequence of this (vectored-thrust / outboard) model is that steering authority scales with thrust: with no thrust the motor produces no force and so no yaw moment, hence essentially no turning -- a trolling motor cannot steer without running. With a bow mount (thruster_x_m > 0) positive steering turns the boat to starboard (heading increases). Cross-coupling between sway and yaw in the damping / added-mass matrices makes the boat visibly "crab" (sway) during a turn.

Integration is semi-implicit Euler: solve for nu_dot, advance nu, then update heading by r*dt and the NED position from the absolute body velocity rotated into the local tangent plane (current/wind already live inside nu).

 73@dataclass
 74class FossenParams:
 75    """Physical constants for the 3-DOF model, tuned to a 4.1 m boat driven by a
 76    single steerable trolling motor mounted at the bow (~1.6 m/s top speed,
 77    ~12-25 deg/s full-thrust/full-steer turn rate).
 78
 79    Masses are in kg, lengths in m, the yaw inertia in kg*m^2. Damping
 80    coefficients are in SI units consistent with forces in N and moments in
 81    N*m acting on velocities in m/s and rad/s.
 82
 83    The surge linear-damping coefficient is *derived* from ``max_thrust_n`` and
 84    ``max_speed_mps`` so that full thrust converges to ~``max_speed_mps``; the
 85    yaw inertia is derived from the hull geometry. See :meth:`__post_init__`.
 86    """
 87
 88    # Boat geometry / mass ----------------------------------------------- #
 89    length: float = 4.1  # overall length (m)
 90    beam: float = 1.7  # beam (m)
 91    mass: float = 300.0  # kg, hull + motor + battery + 1 person
 92
 93    # Thruster -----------------------------------------------------------#
 94    max_thrust_n: float = 250.0  # ~55 lbf trolling motor at full thrust (N)
 95    reverse_efficiency: float = 0.6  # reverse thrust as a fraction of forward
 96    max_speed_mps: float = 1.6  # target top speed at full thrust (m/s)
 97    thruster_x_m: float = 1.7  # CG -> thruster longitudinal distance, + = bow
 98    thruster_y_m: float = 0.0  # CG -> thruster lateral distance, + = starboard
 99    max_steer_angle_deg: float = 35.0  # max motor steer deflection (deg)
100
101    # Added mass (entrained water). Surge is small; sway/yaw are large
102    # because a hull must shove a lot of water sideways. Negative by the
103    # Fossen sign convention. -------------------------------------------- #
104    x_udot: float = -30.0  # added mass in surge
105    y_vdot: float = -250.0  # added mass in sway
106    n_rdot: float = -180.0  # added inertia in yaw
107    y_rdot: float = -40.0  # sway/yaw added-mass coupling
108    n_vdot: float = -40.0  # yaw/sway added-mass coupling
109
110    # Sway / yaw linear damping (surge is derived in __post_init__). ----- #
111    y_v: float = -260.0  # sway drag (large: hull resists sideways motion)
112    n_r: float = -700.0  # yaw drag: sets the sustained turn rate
113    y_r: float = -40.0  # sway/yaw damping coupling
114    n_v: float = -40.0  # yaw/sway damping coupling
115
116    # Quadratic damping (grows with speed^2). ---------------------------- #
117    x_uu: float = -20.0
118    y_vv: float = -180.0
119    n_rr: float = -200.0
120
121    # Hull character / tracking (directional stability). 1.0 = the tuned skiff
122    # (default), ~0.35 = a flat-bottom jon boat (skittish, easily yawed, lots of
123    # leeway), ~2.5 = a deep-V / keelboat (tracks straight, resists turning).
124    # See __post_init__: at this default and the default L/B the multiplier is
125    # exactly 1.0, so the boat is byte-identical to before this knob existed.
126    hull_tracking: float = 1.0
127
128    def __post_init__(self) -> None:
129        # Yaw inertia from a uniform rectangle (Iz = m/12 * (L^2 + B^2)).
130        self.iz: float = self.mass / 12.0 * (self.length**2 + self.beam**2)
131
132        # --- Hull character / tracking (directional stability) ------------- #
133        # A longer, narrower hull tracks better; a short, beamy one is loose.
134        # The slenderness factor is normalised to the default L/B (4.1/1.7) so
135        # it is exactly 1.0 there, and clamped to a sane band. ``hull_tracking``
136        # (clamped) scales it: ~0.35 jon boat .. 1.0 skiff .. ~2.5 keel/deep-V.
137        ht = min(3.0, max(0.25, self.hull_tracking))
138        ref_slender = 4.1 / 1.7
139        slender = min(1.6, max(0.7, (self.length / self.beam) / ref_slender))
140        k = ht * slender
141        # Scale the *directional* coefficients: yaw damping (sustained turn rate
142        # + directional stability) and sway damping (lateral resistance vs
143        # leeway). Higher k => slower turns, less leeway, straighter tracking.
144        self.n_r *= k
145        self.n_rr *= k
146        self.y_v *= k
147        self.y_vv *= k
148        # Expose the realised multiplier for telemetry/tests/introspection.
149        self.hull_k: float = k
150
151        # Derive the surge *linear* drag so that, at top speed, the total
152        # surge drag balances full thrust:
153        #     max_thrust_n = (-x_u) * v_max + (-x_uu) * v_max^2
154        # =>  -x_u = max_thrust_n / v_max - (-x_uu) * v_max
155        v_max = self.max_speed_mps
156        x_u_mag = self.max_thrust_n / v_max - (-self.x_uu) * v_max
157        # Guard against a (mis)configured quadratic term overwhelming the
158        # balance and producing a non-physical negative linear drag.
159        self.x_u: float = -max(x_u_mag, 1.0)
160
161        # Above-water windage areas for the aerodynamic wind force, derived from
162        # geometry assuming a low skiff freeboard (~0.35 m frontal, ~0.45 m
163        # lateral profile). Defaults: A_F≈0.6 m^2, A_L≈1.85 m^2 for the 4.1 m
164        # boat -- matching typical small-craft wind-tunnel reference areas.
165        self.area_front: float = self.beam * 0.35
166        self.area_lateral: float = self.length * 0.45

169class FossenBoat:
170    """A 3-DOF surge-sway-yaw boat. Drop-in for :class:`~vanchor.sim.boat.Boat`."""
171
172    def __init__(
173        self, state: BoatState | None = None, params: FossenParams | None = None
174    ) -> None:
175        self.state = state or BoatState()
176        self.params = params or FossenParams()
177        # Body-frame velocity nu = [u (surge), v (sway), r (yaw rate)].
178        self._nu = np.zeros(3, dtype=float)
179        self._build_matrices()
180
181    # ------------------------------------------------------------------ #
182    # Matrix assembly
183    # ------------------------------------------------------------------ #
184    def _build_matrices(self) -> None:
185        p = self.params
186
187        # Rigid-body mass/inertia matrix (origin at the centre of gravity, so
188        # the surge/sway and yaw blocks decouple).
189        m_rb = np.diag([p.mass, p.mass, p.iz])
190
191        # Added mass matrix.
192        m_a = np.array(
193            [
194                [-p.x_udot, 0.0, 0.0],
195                [0.0, -p.y_vdot, -p.y_rdot],
196                [0.0, -p.n_vdot, -p.n_rdot],
197            ],
198            dtype=float,
199        )
200
201        # Added mass must be symmetric (Fossen: M_A = M_Aᵀ for a port/starboard
202        # symmetric hull). Enforce it so a user setting y_rdot ≠ n_vdot can't make
203        # the model physically inconsistent (and so C(ν) via m2c stays exact).
204        m_a = 0.5 * (m_a + m_a.T)
205
206        self._mass_matrix = m_rb + m_a
207        self._mass_inv = np.linalg.inv(self._mass_matrix)
208
209        # Linear damping (positive-definite resistance, hence the negation of
210        # the conventionally-negative coefficients).
211        self._d_lin = -np.array(
212            [
213                [p.x_u, 0.0, 0.0],
214                [0.0, p.y_v, p.y_r],
215                [0.0, p.n_v, p.n_r],
216            ],
217            dtype=float,
218        )
219
220    def _damping(self, nu: np.ndarray) -> np.ndarray:
221        """Total damping matrix at velocity ``nu`` (linear + quadratic)."""
222        p = self.params
223        u, v, r = nu
224        d_quad = -np.diag(
225            [
226                p.x_uu * abs(u),
227                p.y_vv * abs(v),
228                p.n_rr * abs(r),
229            ]
230        )
231        return self._d_lin + d_quad
232
233    def _tau(self, command: MotorCommand) -> np.ndarray:
234        """Generalized body force/moment ``[X, Y, N]`` from the command.
235
236        A single steerable thruster mounted at longitudinal offset
237        ``thruster_x_m`` from the CG. Thrust magnitude ``T`` is directed along
238        the steered axis ``delta``; the yaw moment is the sway force times the
239        mount lever arm. With a bow mount (``thruster_x_m > 0``) positive
240        steering yields a positive (starboard) yaw moment. With zero thrust the
241        force -- and therefore the yaw moment -- vanishes.
242        """
243        p = self.params
244        cmd = command.clamped()
245        # Trolling-motor props bite less in reverse than forward.
246        eff = 1.0 if cmd.thrust >= 0 else p.reverse_efficiency
247        thrust = cmd.thrust * eff * p.max_thrust_n
248        delta = cmd.steering * math.radians(p.max_steer_angle_deg)
249        fx = thrust * math.cos(delta)
250        fy = thrust * math.sin(delta)
251        # N = x*F_lat - y*F_fwd: the longitudinal arm turns sway into yaw, and a
252        # lateral offset turns the forward thrust into a yaw bias -- so an
253        # off-centre motor yaws the boat even at zero steering.
254        yaw_moment = p.thruster_x_m * fy - p.thruster_y_m * fx
255        return np.array([fx, fy, yaw_moment], dtype=float)
256
257    def _coriolis(self, nu: np.ndarray) -> np.ndarray:
258        """Coriolis-centripetal matrix C(ν) = C_RB + C_A.
259
260        NOT the planetary (Earth-rotation) Coriolis effect -- that is ~0.06 N for
261        this boat and is correctly ignored. This is the *body-frame* Coriolis-
262        centripetal coupling: the fictitious-force terms that arise from writing
263        the dynamics in the hull-fixed frame (which rotates as the boat yaws),
264        plus the added-mass "Munk moment" (C_A). It scales with the boat's OWN
265        motion (u·r, v·r), so it matters precisely when the boat is turning --
266        regardless of vessel size.
267
268        Computed by Fossen's ``m2c`` 3-DOF formula on the full (symmetric) mass
269        matrix -- the same term the otter USV uses. Skew-symmetric (νᵀC ν ≡ 0).
270        This is the centripetal sway/yaw coupling that makes the boat crab
271        *into* a turn; omitting it (as before) inverted the crab and overstated
272        sway by ~4× in hard turns.
273        """
274        m = self._mass_matrix
275        u, v, r = nu
276        c02 = -m[1, 1] * v - m[1, 2] * r
277        c12 = m[0, 0] * u
278        return np.array(
279            [[0.0, 0.0, c02], [0.0, 0.0, c12], [-c02, -c12, 0.0]], dtype=float
280        )
281
282    def _current_body(self, env: Environment, heading_deg: float) -> np.ndarray:
283        """The water current expressed in the body frame ``[u_c, v_c, 0]``.
284
285        The hull is advected by the water, so the hydrodynamic forces act on the
286        velocity *relative to the water* ν_r = ν − ν_c (not the ground velocity).
287        """
288        if env.current_speed == 0.0:
289            return np.zeros(3, dtype=float)
290        ce = env.current_speed * math.sin(math.radians(env.current_dir))
291        cn = env.current_speed * math.cos(math.radians(env.current_dir))
292        h = math.radians(heading_deg)
293        u_c = cn * math.cos(h) + ce * math.sin(h)
294        v_c = -cn * math.sin(h) + ce * math.cos(h)
295        return np.array([u_c, v_c, 0.0], dtype=float)
296
297    def _tau_wind(self, env: Environment, nu: np.ndarray, heading_deg: float) -> np.ndarray:
298        """Aerodynamic wind force/moment ``[X, Y, N]`` on the above-water hull.
299
300        Quadratic in the *apparent* (relative) wind: τ_wind = ½ρ_air·C(γ)·A·V_rw²
301        (Fossen ch. 10 / OCIMF). Unlike a fixed leeway fraction this is
302        heading-dependent, produces a yaw moment (weathervaning -- the dominant
303        disturbance a heading/anchor-hold autopilot must reject), grows with V²,
304        and exerts a real force the thruster must fight. Leeway then *emerges*
305        from this sway force balanced against the hull's sway damping.
306        """
307        p = self.params
308        vw = env.wind_speed
309        if vw <= 0.0:
310            return np.zeros(3, dtype=float)
311        psi = math.radians(heading_deg)
312        bw = math.radians(env.wind_dir)  # "toward which the wind pushes"
313        wu = vw * math.cos(bw - psi)  # wind velocity in the body frame
314        wv = vw * math.sin(bw - psi)
315        # Apparent wind = wind velocity minus the boat's own motion. The drag
316        # force acts ALONG it (downwind), so gw is its body-frame angle (no sign
317        # flip -- the cross-checked beam-on magnitude is ½ρ·cy·A_L·V² ≈ 54 N at
318        # 7 m/s, and a beam wind gives zero yaw).
319        aw_u = wu - nu[0]
320        aw_v = wv - nu[1]
321        vrw = math.hypot(aw_u, aw_v)
322        gw = math.atan2(aw_v, aw_u)
323        q = 0.5 * RHO_AIR * vrw * vrw
324        x = q * WIND_CX * math.cos(gw) * p.area_front
325        y = q * WIND_CY * math.sin(gw) * p.area_lateral
326        n = q * WIND_CN * math.sin(2.0 * gw) * p.area_lateral * p.length
327        return np.array([x, y, n], dtype=float)
328
329    # ------------------------------------------------------------------ #
330    # Integration
331    # ------------------------------------------------------------------ #
332    def step(self, dt: float, command: MotorCommand, env: Environment) -> None:
333        if dt <= 0:
334            return
335        s = self.state
336        nu = self._nu
337
338        # Current advects the hull: the hydrodynamic forces (Coriolis + damping)
339        # act on the velocity through the WATER, nu_r = nu - nu_c, not the ground
340        # velocity. Wind enters as an aerodynamic force on tau. Position then
341        # integrates the ABSOLUTE velocity nu, so the boat genuinely drifts with
342        # the water (nu relaxes toward nu_c under drag) -- no kinematic offset.
343        nu_c = self._current_body(env, s.heading_deg)
344        nu_r = nu - nu_c
345
346        # Solve M*nu_dot + C(nu_r)*nu_r + D(nu_r)*nu_r = tau_thrust + tau_wind.
347        tau = self._tau(command) + self._tau_wind(env, nu, s.heading_deg)
348        cor = self._coriolis(nu_r)
349        damping = self._damping(nu_r)
350        nu_dot = self._mass_inv @ (tau - cor @ nu_r - damping @ nu_r)
351
352        # Semi-implicit Euler: advance the velocities first.
353        nu = nu + nu_dot * dt
354        self._nu = nu
355        u, v, r = nu
356
357        # Heading from the yaw rate (r is rad/s).
358        s.heading_deg = normalize_deg(s.heading_deg + math.degrees(r) * dt)
359
360        # Rotate the absolute body velocity into NED -> velocity over ground.
361        # Heading 0 = north, +heading = clockwise (toward east); +sway (v) is to
362        # starboard. Current/wind already live inside nu, so nothing is added.
363        h = math.radians(s.heading_deg)
364        sin_h, cos_h = math.sin(h), math.cos(h)
365        north = u * cos_h - v * sin_h
366        east = u * sin_h + v * cos_h
367        s.ground_ve = east
368        s.ground_vn = north
369        s.point = offset_meters(s.point, east * dt, north * dt)
370
371        # Forward speed THROUGH THE WATER (|nu_r|) for the rest of the system;
372        # speed-over-ground = hypot(ground_ve, ground_vn) and differs when a
373        # current flows.
374        s.speed_mps = math.hypot(u - nu_c[0], v - nu_c[1])
375        s.timestamp += dt
376
377    def teleport(self, point: GeoPoint, heading: float | None = None) -> None:
378        """Instantly move ground truth to ``point`` (optionally set heading) and
379        zero the body-frame velocities (surge/sway/yaw) so the boat stops dead
380        instead of coasting from its pre-teleport momentum."""
381        s = self.state
382        s.point = point
383        if heading is not None:
384            s.heading_deg = normalize_deg(float(heading))
385        self._nu = np.zeros(3, dtype=float)
386        s.speed_mps = 0.0
387        s.ground_ve = 0.0
388        s.ground_vn = 0.0
389
390    def truth(self) -> BoatState:
391        """An immutable-ish snapshot of the current ground truth."""
392        s = self.state
393        return BoatState(
394            point=s.point,
395            heading_deg=s.heading_deg,
396            speed_mps=s.speed_mps,
397            timestamp=s.timestamp,
398            ground_ve=s.ground_ve,
399            ground_vn=s.ground_vn,
400        )

129    def __post_init__(self) -> None:
130        # Yaw inertia from a uniform rectangle (Iz = m/12 * (L^2 + B^2)).
131        self.iz: float = self.mass / 12.0 * (self.length**2 + self.beam**2)

134        # A longer, narrower hull tracks better; a short, beamy one is loose.
135        # The slenderness factor is normalised to the default L/B (4.1/1.7) so
136        # it is exactly 1.0 there, and clamped to a sane band. ``hull_tracking``
137        # (clamped) scales it: ~0.35 jon boat .. 1.0 skiff .. ~2.5 keel/deep-V.
138        ht = min(3.0, max(0.25, self.hull_tracking))
139        ref_slender = 4.1 / 1.7
140        slender = min(1.6, max(0.7, (self.length / self.beam) / ref_slender))
141        k = ht * slender
142        # Scale the *directional* coefficients: yaw damping (sustained turn rate
143        # + directional stability) and sway damping (lateral resistance vs
144        # leeway). Higher k => slower turns, less leeway, straighter tracking.
145        self.n_r *= k
146        self.n_rr *= k
147        self.y_v *= k
148        self.y_vv *= k
149        # Expose the realised multiplier for telemetry/tests/introspection.
150        self.hull_k: float = k

152        # Derive the surge *linear* drag so that, at top speed, the total
153        # surge drag balances full thrust:
154        #     max_thrust_n = (-x_u) * v_max + (-x_uu) * v_max^2
155        # =>  -x_u = max_thrust_n / v_max - (-x_uu) * v_max
156        v_max = self.max_speed_mps
157        x_u_mag = self.max_thrust_n / v_max - (-self.x_uu) * v_max
158        # Guard against a (mis)configured quadratic term overwhelming the
159        # balance and producing a non-physical negative linear drag.
160        self.x_u: float = -max(x_u_mag, 1.0)

162        # Above-water windage areas for the aerodynamic wind force, derived from
163        # geometry assuming a low skiff freeboard (~0.35 m frontal, ~0.45 m
164        # lateral profile). Defaults: A_F≈0.6 m^2, A_L≈1.85 m^2 for the 4.1 m
165        # boat -- matching typical small-craft wind-tunnel reference areas.
166        self.area_front: float = self.beam * 0.35
167        self.area_lateral: float = self.length * 0.45

185    def _build_matrices(self) -> None:
186        p = self.params
187
188        # Rigid-body mass/inertia matrix (origin at the centre of gravity, so
189        # the surge/sway and yaw blocks decouple).
190        m_rb = np.diag([p.mass, p.mass, p.iz])
191
192        # Added mass matrix.
193        m_a = np.array(
194            [
195                [-p.x_udot, 0.0, 0.0],
196                [0.0, -p.y_vdot, -p.y_rdot],
197                [0.0, -p.n_vdot, -p.n_rdot],
198            ],
199            dtype=float,
200        )
201
202        # Added mass must be symmetric (Fossen: M_A = M_Aᵀ for a port/starboard
203        # symmetric hull). Enforce it so a user setting y_rdot ≠ n_vdot can't make
204        # the model physically inconsistent (and so C(ν) via m2c stays exact).
205        m_a = 0.5 * (m_a + m_a.T)
206
207        self._mass_matrix = m_rb + m_a
208        self._mass_inv = np.linalg.inv(self._mass_matrix)
209
210        # Linear damping (positive-definite resistance, hence the negation of
211        # the conventionally-negative coefficients).
212        self._d_lin = -np.array(
213            [
214                [p.x_u, 0.0, 0.0],
215                [0.0, p.y_v, p.y_r],
216                [0.0, p.n_v, p.n_r],
217            ],
218            dtype=float,
219        )

221    def _damping(self, nu: np.ndarray) -> np.ndarray:
222        """Total damping matrix at velocity ``nu`` (linear + quadratic)."""
223        p = self.params
224        u, v, r = nu
225        d_quad = -np.diag(
226            [
227                p.x_uu * abs(u),
228                p.y_vv * abs(v),
229                p.n_rr * abs(r),
230            ]
231        )
232        return self._d_lin + d_quad

234    def _tau(self, command: MotorCommand) -> np.ndarray:
235        """Generalized body force/moment ``[X, Y, N]`` from the command.
236
237        A single steerable thruster mounted at longitudinal offset
238        ``thruster_x_m`` from the CG. Thrust magnitude ``T`` is directed along
239        the steered axis ``delta``; the yaw moment is the sway force times the
240        mount lever arm. With a bow mount (``thruster_x_m > 0``) positive
241        steering yields a positive (starboard) yaw moment. With zero thrust the
242        force -- and therefore the yaw moment -- vanishes.
243        """
244        p = self.params
245        cmd = command.clamped()
246        # Trolling-motor props bite less in reverse than forward.
247        eff = 1.0 if cmd.thrust >= 0 else p.reverse_efficiency
248        thrust = cmd.thrust * eff * p.max_thrust_n
249        delta = cmd.steering * math.radians(p.max_steer_angle_deg)
250        fx = thrust * math.cos(delta)
251        fy = thrust * math.sin(delta)
252        # N = x*F_lat - y*F_fwd: the longitudinal arm turns sway into yaw, and a
253        # lateral offset turns the forward thrust into a yaw bias -- so an
254        # off-centre motor yaws the boat even at zero steering.
255        yaw_moment = p.thruster_x_m * fy - p.thruster_y_m * fx
256        return np.array([fx, fy, yaw_moment], dtype=float)
257
258    def _coriolis(self, nu: np.ndarray) -> np.ndarray:
259        """Coriolis-centripetal matrix C(ν) = C_RB + C_A.
260
261        NOT the planetary (Earth-rotation) Coriolis effect -- that is ~0.06 N for
262        this boat and is correctly ignored. This is the *body-frame* Coriolis-
263        centripetal coupling: the fictitious-force terms that arise from writing
264        the dynamics in the hull-fixed frame (which rotates as the boat yaws),
265        plus the added-mass "Munk moment" (C_A). It scales with the boat's OWN
266        motion (u·r, v·r), so it matters precisely when the boat is turning --
267        regardless of vessel size.
268
269        Computed by Fossen's ``m2c`` 3-DOF formula on the full (symmetric) mass
270        matrix -- the same term the otter USV uses. Skew-symmetric (νᵀC ν ≡ 0).
271        This is the centripetal sway/yaw coupling that makes the boat crab
272        *into* a turn; omitting it (as before) inverted the crab and overstated
273        sway by ~4× in hard turns.
274        """
275        m = self._mass_matrix
276        u, v, r = nu
277        c02 = -m[1, 1] * v - m[1, 2] * r
278        c12 = m[0, 0] * u
279        return np.array(
280            [[0.0, 0.0, c02], [0.0, 0.0, c12], [-c02, -c12, 0.0]], dtype=float
281        )

283    def _current_body(self, env: Environment, heading_deg: float) -> np.ndarray:
284        """The water current expressed in the body frame ``[u_c, v_c, 0]``.
285
286        The hull is advected by the water, so the hydrodynamic forces act on the
287        velocity *relative to the water* ν_r = ν − ν_c (not the ground velocity).
288        """
289        if env.current_speed == 0.0:
290            return np.zeros(3, dtype=float)
291        ce = env.current_speed * math.sin(math.radians(env.current_dir))
292        cn = env.current_speed * math.cos(math.radians(env.current_dir))
293        h = math.radians(heading_deg)
294        u_c = cn * math.cos(h) + ce * math.sin(h)
295        v_c = -cn * math.sin(h) + ce * math.cos(h)
296        return np.array([u_c, v_c, 0.0], dtype=float)
297
298    def _tau_wind(self, env: Environment, nu: np.ndarray, heading_deg: float) -> np.ndarray:
299        """Aerodynamic wind force/moment ``[X, Y, N]`` on the above-water hull.
300
301        Quadratic in the *apparent* (relative) wind: τ_wind = ½ρ_air·C(γ)·A·V_rw²
302        (Fossen ch. 10 / OCIMF). Unlike a fixed leeway fraction this is
303        heading-dependent, produces a yaw moment (weathervaning -- the dominant
304        disturbance a heading/anchor-hold autopilot must reject), grows with V²,
305        and exerts a real force the thruster must fight. Leeway then *emerges*
306        from this sway force balanced against the hull's sway damping.
307        """
308        p = self.params
309        vw = env.wind_speed
310        if vw <= 0.0:
311            return np.zeros(3, dtype=float)
312        psi = math.radians(heading_deg)
313        bw = math.radians(env.wind_dir)  # "toward which the wind pushes"
314        wu = vw * math.cos(bw - psi)  # wind velocity in the body frame
315        wv = vw * math.sin(bw - psi)
316        # Apparent wind = wind velocity minus the boat's own motion. The drag
317        # force acts ALONG it (downwind), so gw is its body-frame angle (no sign
318        # flip -- the cross-checked beam-on magnitude is ½ρ·cy·A_L·V² ≈ 54 N at
319        # 7 m/s, and a beam wind gives zero yaw).
320        aw_u = wu - nu[0]
321        aw_v = wv - nu[1]
322        vrw = math.hypot(aw_u, aw_v)
323        gw = math.atan2(aw_v, aw_u)
324        q = 0.5 * RHO_AIR * vrw * vrw
325        x = q * WIND_CX * math.cos(gw) * p.area_front
326        y = q * WIND_CY * math.sin(gw) * p.area_lateral
327        n = q * WIND_CN * math.sin(2.0 * gw) * p.area_lateral * p.length
328        return np.array([x, y, n], dtype=float)

333    def step(self, dt: float, command: MotorCommand, env: Environment) -> None:
334        if dt <= 0:
335            return
336        s = self.state
337        nu = self._nu
338
339        # Current advects the hull: the hydrodynamic forces (Coriolis + damping)
340        # act on the velocity through the WATER, nu_r = nu - nu_c, not the ground
341        # velocity. Wind enters as an aerodynamic force on tau. Position then
342        # integrates the ABSOLUTE velocity nu, so the boat genuinely drifts with
343        # the water (nu relaxes toward nu_c under drag) -- no kinematic offset.
344        nu_c = self._current_body(env, s.heading_deg)
345        nu_r = nu - nu_c
346
347        # Solve M*nu_dot + C(nu_r)*nu_r + D(nu_r)*nu_r = tau_thrust + tau_wind.
348        tau = self._tau(command) + self._tau_wind(env, nu, s.heading_deg)
349        cor = self._coriolis(nu_r)
350        damping = self._damping(nu_r)
351        nu_dot = self._mass_inv @ (tau - cor @ nu_r - damping @ nu_r)
352
353        # Semi-implicit Euler: advance the velocities first.
354        nu = nu + nu_dot * dt
355        self._nu = nu
356        u, v, r = nu
357
358        # Heading from the yaw rate (r is rad/s).
359        s.heading_deg = normalize_deg(s.heading_deg + math.degrees(r) * dt)
360
361        # Rotate the absolute body velocity into NED -> velocity over ground.
362        # Heading 0 = north, +heading = clockwise (toward east); +sway (v) is to
363        # starboard. Current/wind already live inside nu, so nothing is added.
364        h = math.radians(s.heading_deg)
365        sin_h, cos_h = math.sin(h), math.cos(h)
366        north = u * cos_h - v * sin_h
367        east = u * sin_h + v * cos_h
368        s.ground_ve = east
369        s.ground_vn = north
370        s.point = offset_meters(s.point, east * dt, north * dt)
371
372        # Forward speed THROUGH THE WATER (|nu_r|) for the rest of the system;
373        # speed-over-ground = hypot(ground_ve, ground_vn) and differs when a
374        # current flows.
375        s.speed_mps = math.hypot(u - nu_c[0], v - nu_c[1])
376        s.timestamp += dt
