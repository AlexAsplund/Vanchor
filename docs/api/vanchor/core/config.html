vanchor.core.config

Typed, nested, file-backed application configuration.

The whole controller is configured from a single AppConfig tree of small dataclasses. Each sub-config maps onto the constructors of an existing component (the simulator, sensors, controller, control modes, helm and the environment) so the integrator can wire things up by reading fields straight off the config rather than threading loose keyword arguments around.

Configs can be loaded from a YAML (.yaml/.yml) or JSON (.json) file. Unknown keys are ignored and any missing key falls back to its default, so partial config files are always valid. With no path (or a missing file) the built-in defaults are returned unchanged.

 43@dataclass
 44class BoatConfig:
 45    """Physical boat + trolling-motor geometry.
 46
 47    Feeds the ``fossen`` 3-DOF physics model (which is bow-mount aware) and the
 48    ``simple`` model's speed/turn limits. ``thruster_mount`` captures *where* the
 49    steerable trolling motor sits: a bow mount pulls the bow around, a stern
 50    mount pushes it -- the sign of the resulting yaw is opposite, which the
 51    model accounts for via the longitudinal offset from the centre of gravity.
 52    """
 53
 54    length_m: float = 4.1
 55    beam_m: float = 1.7
 56    mass_kg: float = 300.0
 57    max_speed_mps: float = 1.6
 58    max_thrust_n: float = 250.0  # ~55 lbf trolling motor
 59    reverse_efficiency: float = 0.6  # reverse prop thrust as a fraction of forward
 60    thruster_mount: str = "bow"  # "bow" | "stern" | "center"
 61    thruster_offset_m: float | None = None  # explicit CG->thruster (+fwd); overrides mount
 62    thruster_y_m: float = 0.0  # lateral CG->thruster offset (+ = starboard)
 63    # Longitudinal CG position, as a fraction of length AFT of the geometric
 64    # centre. Real boats carry their mass aft (battery, fuel, outboard/engine,
 65    # the helm seat), so the CG sits behind centre -- which LENGTHENS the lever
 66    # arm from the CG to a bow motor (sharper turns) and shortens it for a stern
 67    # motor. 0 = CG at centre (old behaviour); ~0.1 is typical. Folded into
 68    # thruster_x_m() below (and thus the Fossen yaw moment).
 69    cg_aft_frac: float = 0.10
 70    # Thrust-yaw feed-forward: a steering deflection that pre-cancels the yaw a
 71    # laterally-offset thruster induces under straight thrust. None = derive from
 72    # geometry (atan2(thruster_y_m, |thruster_x_m|)); a number overrides it. A
 73    # calibration-measured TRIM (radians) is added on top of whichever is used.
 74    thrust_yaw_ff: float | None = None
 75    thrust_yaw_ff_trim: float = 0.0  # calibration refinement (radians) on the FF angle
 76    max_steer_angle_deg: float = 180.0  # full mechanical swing (manual reaches this)
 77    autopilot_steer_deg: float = 35.0  # authority the autopilot actually uses
 78    max_steer_rate_dps: float = 50.0  # how fast the steering head can rotate (deg/s)
 79    max_turn_rate_deg: float = 18.0  # used by the kinematic "simple" model
 80    # Hull character / tracking (directional stability) for the fossen model.
 81    # ~0.35 = jon boat (flat-bottom: loose, skittish, lots of leeway), 1.0 =
 82    # current skiff (default), ~2.5 = deep-V / keel (tracks straight, resists
 83    # turning). Clamped to ~0.25..3.0 on use. Scales the yaw + sway damping (see
 84    # FossenParams.__post_init__); at 1.0 with the default L/B it is a no-op.
 85    hull_tracking: float = 1.0
 86    # Steering gearbox (the closed-loop azimuth unit; see cad/steering.py).
 87    shaft_dia_mm: float = 25.4
 88    steer_range_deg: float = 185.0  # +/- cable-wrap mechanical limit of the head
 89    steer_reduction: float = 4.0  # pinion->ring reduction
 90    # Sonar transducer beam (cone) angle in degrees. NMEA DPT/DBT carry only a
 91    # depth, never a beam angle, so this configurable default is what sizes the
 92    # depth-map footprint: footprint diameter at depth d = 2*d*tan(cone/2).
 93    sonar_cone_deg: float = 20.0
 94
 95    def thruster_x_m(self) -> float:
 96        """Signed longitudinal distance from CG to the thruster (+ = forward).
 97
 98        The mount fixes the thruster's position relative to the hull's geometric
 99        centre (bow ~+0.42L, stern ~-0.42L); the CG sits ``cg_aft_frac`` of a
100        length BEHIND that centre, so the CG->thruster arm is the gap between
101        them. An explicit ``thruster_offset_m`` already encodes the CG distance
102        and is used verbatim.
103        """
104        if self.thruster_offset_m is not None:
105            return self.thruster_offset_m
106        frac = {"bow": 0.42, "stern": -0.42, "center": 0.0}.get(self.thruster_mount, 0.42)
107        return frac * self.length_m + self.cg_aft_frac * self.length_m
108
109    def thrust_yaw_ff_angle(self) -> float:
110        """Feed-forward steering deflection (radians) that cancels the straight-
111        thrust yaw of a laterally-offset thruster.
112
113        A thruster at ``(x, y)`` making forward thrust ``F`` and steering-induced
114        lateral force produces yaw ``N = x*F_lat - y*F_fwd``. Deflecting the motor
115        by ``delta`` gives ``F_fwd = F*cos(delta)``, ``F_lat = F*sin(delta)`` so
116        ``N = F*(x*sin(delta) - y*cos(delta))`` which is zero when
117        ``x*sin(delta) = y*cos(delta)`` => ``delta = atan2(y, |x|)`` -- independent
118        of thrust magnitude. ``thrust_yaw_ff`` overrides this geometric value; a
119        measured ``thrust_yaw_ff_trim`` is then added on top.
120
121        Note the lever arm uses ``|x|``: a stern mount (x < 0) needs the same
122        *physical* deflection sign as a bow mount to oppose the same lateral
123        offset; the bow/stern steering-authority flip is handled separately by the
124        helm's ``steer_sign``.
125        """
126        if self.thrust_yaw_ff is not None:
127            return self.thrust_yaw_ff + self.thrust_yaw_ff_trim
128        x_mag = abs(self.thruster_x_m())
129        return math.atan2(self.thruster_y_m, x_mag) + self.thrust_yaw_ff_trim

263class HardwareConfig:
264    """Real serial hardware. ``enabled`` is the master switch (False = full
265    simulation, the default; True = all real serial devices).
266
267    Per-device ``*_source`` overrides let you **mix** simulated and real devices
268    — e.g. drive a real steering servo while the boat itself is simulated, to

280    # e.g. a phone or chart-plotter GPS. So "GPS from NMEA" is never blocked.
281    gps_source: str | None = None      # "sim" | "serial" | "nmea"
282    compass_source: str | None = None  # "sim" | "serial" | "nmea"
283    depth_source: str | None = None    # "sim" | "nmea" (no serial depth yet)
284    motor_source: str | None = None    # "sim" | "serial" | "both"

286    # Per-device settings from a driver's device_menu(), keyed by device kind:
287    # e.g. {"compass": {"declination_mode": "manual", "manual_declination_deg": 3}}.

310@dataclass
311class AppConfig:
312    """The root configuration tree."""
313
314    data_dir: str = "vanchor_data"  # persisted depth map + debug recordings
315    sim: SimConfig = field(default_factory=SimConfig)
316    boat: BoatConfig = field(default_factory=BoatConfig)
317    environment: EnvironmentConfig = field(default_factory=EnvironmentConfig)
318    sensors: SensorConfig = field(default_factory=SensorConfig)
319    control: ControlConfig = field(default_factory=ControlConfig)
320    safety: SafetyConfig = field(default_factory=SafetyConfig)
321    battery: BatteryConfig = field(default_factory=BatteryConfig)
322    server: ServerConfig = field(default_factory=ServerConfig)
323    hardware: HardwareConfig = field(default_factory=HardwareConfig)
324    nmea_tcp: NmeaTcpConfig = field(default_factory=NmeaTcpConfig)
325
326    @classmethod
327    def from_dict(cls, data: dict[str, Any] | None) -> "AppConfig":
328        """Build an :class:`AppConfig` from a (possibly partial) mapping.
329
330        Performs a deep, defensive merge: unknown keys are ignored and any key
331        not present keeps its default. ``None`` is treated as an empty mapping.
332        """
333        data = data or {}
334        kwargs: dict[str, Any] = {}
335        for f in fields(cls):
336            if f.name not in _SUBCONFIGS:
337                # plain scalar field (e.g. data_dir): take value or keep default.
338                if f.name in data:
339                    kwargs[f.name] = data[f.name]
340                continue
341            sub_cls = f.type if isinstance(f.type, type) else _SUBCONFIGS[f.name]
342            kwargs[f.name] = _build_sub(sub_cls, data.get(f.name))
343        return cls(**kwargs)
344
345    def to_dict(self) -> dict[str, Any]:
346        """Serialize the full tree to plain nested dicts."""
347        return asdict(self)

350# Mapping of AppConfig field name -> its sub-config dataclass. Used by
351# ``from_dict`` because ``from __future__ import annotations`` turns field
352# type annotations into strings.
353_SUBCONFIGS: dict[str, type] = {
354    "sim": SimConfig,
355    "boat": BoatConfig,
356    "environment": EnvironmentConfig,
357    "sensors": SensorConfig,
358    "control": ControlConfig,
359    "safety": SafetyConfig,
360    "battery": BatteryConfig,
361    "server": ServerConfig,
362    "hardware": HardwareConfig,
363    "nmea_tcp": NmeaTcpConfig,
364}

367def _build_sub(sub_cls: type, data: Any) -> Any:
368    """Instantiate a leaf sub-config, ignoring unknown keys and filling
369    defaults for anything absent."""
370    if not isinstance(data, dict):
371        return sub_cls()
372    known = {f.name for f in fields(sub_cls)}
373    kwargs = {k: v for k, v in data.items() if k in known}
374    return sub_cls(**kwargs)

377# --- Persisted, editable device/hardware config (devices.json) ----------- #
378# The device config is the only part of the config that the running UI can
379# edit + persist (separately from the load-only YAML/defaults). It lives in a
380# small ``<data_dir>/devices.json`` so a saved hardware setup survives restarts.
381# Shape: ``{"hardware": {...HardwareConfig fields...},
382#           "nmea_tcp": {...NmeaTcpConfig fields...}}`` -- the same defensive,
383# field-merge tolerance as the rest of the config (unknown/missing keys OK).
384DEVICES_FILE = "devices.json"
385
386
387def _merge_into(obj: Any, data: Any) -> None:
388    """Overwrite the known fields of dataclass ``obj`` from mapping ``data``,
389    coercing numeric/bool fields to the declared type. Unknown/absent keys and
390    ``None`` mean "keep what's there". Mutates ``obj`` in place."""
391    if not isinstance(data, dict):
392        return
393    for f in fields(obj):
394        if f.name not in data or data[f.name] is None:
395            continue
396        val = data[f.name]
397        cur = getattr(obj, f.name)
398        if isinstance(cur, bool):
399            val = bool(val)
400        elif isinstance(cur, int) and not isinstance(cur, bool):
401            val = int(val)
402        elif isinstance(cur, float):
403            val = float(val)
404        setattr(obj, f.name, val)

407def load_device_overrides(data_dir: str | Path) -> dict[str, Any] | None:
408    """Read ``<data_dir>/devices.json`` if present, else ``None``.
409
410    Returns the parsed ``{"hardware": {...}, "nmea_tcp": {...}}`` mapping. A
411    missing file (the common case) or a corrupt/non-mapping file returns
412    ``None`` so startup falls back to the base config untouched.
413    """
414    p = Path(data_dir) / DEVICES_FILE
415    if not p.exists():
416        return None
417    try:
418        data = json.loads(p.read_text(encoding="utf-8"))
419    except (OSError, ValueError) as exc:
420        log.warning("ignoring unreadable %s: %s", p, exc)
421        return None
422    if not isinstance(data, dict):
423        log.warning("ignoring %s: not a mapping", p)
424        return None
425    return data

428def apply_device_overrides(config: AppConfig, data_dir: str | Path | None = None) -> AppConfig:
429    """Override ``config.hardware`` + ``config.nmea_tcp`` from a persisted
430    ``devices.json`` (if one exists under ``data_dir``), then return ``config``.
431
432    ``data_dir`` defaults to ``config.data_dir``. A field-level merge: any saved
433    key overrides the loaded base, missing/extra keys are tolerated. Call this
434    after :func:`load` and before building the runtime so a saved device config

475def load_dotenv(path: str | Path | None = None) -> dict[str, str]:
476    """Read a ``.env`` file and set any of its keys not already in ``os.environ``.
477
478    No external dependency. Parses simple ``KEY=VALUE`` lines, ignoring blank
479    lines and ``#`` comments, stripping surrounding single/double quotes and a
480    leading ``export ``. A real environment variable always wins (existing keys
481    are left untouched). The path defaults to ``$VANCHOR_ENV_FILE`` else
482    ``./.env``; a missing file is a quiet no-op. Returns the parsed mapping.
483    """
484    if path is None:
485        path = os.environ.get("VANCHOR_ENV_FILE", ".env")
486    p = Path(path)
487    if not p.exists():
488        return {}
489    parsed: dict[str, str] = {}
490    for raw in p.read_text(encoding="utf-8").splitlines():
491        line = raw.strip()
492        if not line or line.startswith("#"):
493            continue
494        if line.startswith("export "):
495            line = line[len("export "):].lstrip()
496        if "=" not in line:
497            continue
498        key, _, val = line.partition("=")
499        key = key.strip()
500        if not key:
501            continue
502        val = val.strip()
503        if len(val) >= 2 and val[0] == val[-1] and val[0] in ("'", '"'):
504            val = val[1:-1]
505        parsed[key] = val
506        os.environ.setdefault(key, val)  # real env wins
507    log.info("Loaded %d var(s) from %s", len(parsed), p)
508    return parsed

511def apply_env_overrides(config: AppConfig) -> AppConfig:
512    """Override config fields from ``VANCHOR_*`` env vars (in place) and return it.
513
514    Only set variables are applied (unset ones leave the loaded/default value
515    untouched); each is coerced to the field's type. Booleans use
516    :func:`_parse_bool` ("1/true/yes/on").
517    """
518    env = os.environ
519
520    def _apply(var: str, target: Any, attr: str, coerce: Callable[[str], Any]) -> None:
521        val = env.get(var)
522        if val is None:
523            return
524        try:
525            setattr(target, attr, coerce(val))
526        except (TypeError, ValueError) as exc:
527            log.warning("ignoring %s=%r: %s", var, val, exc)
528
529    # Server.
530    _apply("VANCHOR_HOST", config.server, "host", str)
531    _apply("VANCHOR_PORT", config.server, "port", int)
532    # Data.
533    _apply("VANCHOR_DATA_DIR", config, "data_dir", str)
534    # Simulation.
535    _apply("VANCHOR_MODEL", config.sim, "model", str)
536    _apply("VANCHOR_TIME_SCALE", config.sim, "time_scale", float)
537    _apply("VANCHOR_PHYSICS_HZ", config.sim, "physics_hz", float)
538    _apply("VANCHOR_SIM_START_LAT", config.sim, "start_lat", float)
539    _apply("VANCHOR_SIM_START_LON", config.sim, "start_lon", float)
540    # Sensors & hardware.
541    _apply("VANCHOR_HARDWARE", config.hardware, "enabled", _parse_bool)
542    _apply("VANCHOR_GPS_PORT", config.hardware, "gps_port", str)
543    _apply("VANCHOR_COMPASS_PORT", config.hardware, "compass_port", str)
544    _apply("VANCHOR_MOTOR_PORT", config.hardware, "motor_port", str)
545    _apply("VANCHOR_BAUDRATE", config.hardware, "baudrate", int)
546    _apply("VANCHOR_GPS_SOURCE", config.hardware, "gps_source", str)
547    _apply("VANCHOR_COMPASS_SOURCE", config.hardware, "compass_source", str)
548    _apply("VANCHOR_DEPTH_SOURCE", config.hardware, "depth_source", str)
549    _apply("VANCHOR_MOTOR_SOURCE", config.hardware, "motor_source", str)
550    # NMEA bridge.
551    _apply("VANCHOR_NMEA_TCP", config.nmea_tcp, "enabled", _parse_bool)
552    _apply("VANCHOR_NMEA_TCP_HOST", config.nmea_tcp, "host", str)
553    _apply("VANCHOR_NMEA_TCP_PORT", config.nmea_tcp, "port", int)
554    return config

557def load(path: str | Path | None) -> AppConfig:
558    """Load an :class:`AppConfig` from a ``.yaml``/``.yml``/``.json`` file.
559
560    Returns the built-in defaults when ``path`` is None or the file does not
561    exist. Raises ``ValueError`` for an unsupported extension and propagates
562    parse errors from the underlying loader.
563
564    A ``.env`` file is read first (see :func:`load_dotenv`) and the resulting
565    ``VANCHOR_*`` environment variables are applied *after* the file/defaults so
566    the environment always wins over the YAML/JSON and the built-in defaults.
567    """
568    load_dotenv()
569
570    if path is None:
571        return apply_env_overrides(AppConfig())
572
573    p = Path(path)
574    if not p.exists():
575        log.warning("Config file %s not found; using defaults", p)
576        return apply_env_overrides(AppConfig())
577
578    text = p.read_text(encoding="utf-8")
579    suffix = p.suffix.lower()
580    if suffix in (".yaml", ".yml"):
581        import yaml  # PyYAML, only needed for YAML configs.
582
583        data = yaml.safe_load(text)
584    elif suffix == ".json":
585        data = json.loads(text)
586    else:
587        raise ValueError(f"Unsupported config extension: {p.suffix!r}")
588
589    if data is not None and not isinstance(data, dict):
590        raise ValueError(f"Config file {p} must contain a mapping at the top level")
591
592    log.info("Loaded config from %s", p)
593    return apply_env_overrides(AppConfig.from_dict(data))

130        return math.atan2(self.thruster_y_m, x_mag) + self.thrust_yaw_ff_trim

Feed-forward steering deflection (radians) that cancels the straight- thrust yaw of a laterally-offset thruster.

A thruster at (x, y) making forward thrust F and steering-induced lateral force produces yaw N = x*F_lat - y*F_fwd. Deflecting the motor by delta gives F_fwd = F*cos(delta), F_lat = F*sin(delta) so N = F*(x*sin(delta) - y*cos(delta)) which is zero when x*sin(delta) = y*cos(delta) => delta = atan2(y, |x|) -- independent of thrust magnitude. thrust_yaw_ff overrides this geometric value; a measured thrust_yaw_ff_trim is then added on top.

Note the lever arm uses |x|: a stern mount (x < 0) needs the same physical deflection sign as a bow mount to oppose the same lateral offset; the bow/stern steering-authority flip is handled separately by the helm's steer_sign.

483    ``./.env``; a missing file is a quiet no-op. Returns the parsed mapping.
484    """

488    if not p.exists():

491    for raw in p.read_text(encoding="utf-8").splitlines():
492        line = raw.strip()
493        if not line or line.startswith("#"):
494            continue
